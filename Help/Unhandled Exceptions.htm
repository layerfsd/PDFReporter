Unhandled Exception is an unplanned program error or just an uncaught error. Almost all such errors will be displayed in a dialog allowing reporting error details directly to AxiomCoders Support. Reporting errors is an optional step which will allow AxiomCoders development team to reproduce and fix reported issues and improve overall product quality as soon as possible.

Unhandled Exception report dialog has only one editable field: Description. It is optional, but writing as much details as possible about how to reproduce the error and which actions were taken before the error occurred will help AxiomCoders development team to more quickly duplicate the error and resolve it, providing the fix in the next product version.

 

Details tab is informative only. It contains all the information which will be reported to AxiomCoders or when saving the error details. Nothing else will be recorded and privacy of the user will not be violated in any way.

 

Buttons on the dialog have the following function:

            Report Exception will open the system default e-mail client new message and automatically fill in the To field addressing the e-mail to AxiomCoders Support team, Subject field reporting AxiomCoders product name and version, and insert error details into body of the e-mail. The e-mail will not be sent out automatically – it must be done manually which allows for preview of all the information being sent to AxiomCoders. Note that the From field is usually automatically filled by the e-mail client itself and that sending the e-mail message to AxiomCoders Support team will probably reveal your name and e-mail address so they can contact you for more details about the reported issue or notify you about the issue resolution.
            Copy to Clipboard will copy all error details to the clipboard. This is good when sending error details manually to AxiomCoders Support team as error details can just be pasted wherever needed.
            Save to File will open the standard Windows save file dialog asking for a text (*.txt) file name where to save the error details. It is also good when manually sending error details to AxiomCoders Support team using a single file.
